# Desktop Dashboard And Editor Design

## Goal

把桌面端首页重构成更规整的 dashboard 布局，同时把文档页编辑区重心切到 Markdown 正文编辑，修复预览只显示纯文本的问题。

## Context

当前桌面端已经具备首页、二级页和深色星系视觉语言，但首页仍然偏“堆叠 panel”，缺少明确的 dashboard 层次；文档编辑区虽然有 frontmatter、正文和预览，但输入密度偏大，正文编辑不是最突出的工作区，预览也没有真正解析 Markdown。

用户给出的参考图强调：

- 左侧固定导航，主区顶部有搜索和控制条
- 首页主体是规整的 dashboard grid，而不是单一大横幅
- 各模块层级清晰，卡片视觉一致，留白稳定

同时，当前产品已经建立了深色、结构网格、星系轨道的视觉方向，这部分需要保留，不能直接转成白底 SaaS 风格。

## Design Direction

### 1. 首页改成 Dashboard Shell

首页继续保留左侧导航，但主区切成三层：

- 顶部搜索与动作条
- 轻量总览头部
- 规整模块化 dashboard grid

搜索入口从原来的首页卡片内部上移到顶层，形成真正的主导航入口。总览头部不再承担全部视觉重量，只负责标题、当前页面说明和几项关键指标。其下通过统一栅格承载概览卡、快捷入口、最近操作和仓库摘要。

### 2. 保留深色星系语义，不复制参考图的白底语义

页面结构借鉴参考图，但视觉保持当前产品的品牌方向：

- 深蓝黑主背景
- 结构网格与轨道感
- 蓝色作为结构信息色，橙色作为行动色
- 卡片边界更薄，亮部更克制，减少目前“每块都很重”的问题

目标不是做成标准 HR dashboard，而是做成更有秩序感的“知识星系控制台”。

### 3. 文档页把 Markdown 编辑作为核心工作面

文档页会从“元数据优先”调整为“正文优先”：

- 左侧仍是文档列表
- 右侧编辑器区域切成主编辑列和辅助信息列
- 正文编辑器占最大面积
- frontmatter 字段收紧为更小的辅助控件
- 预览区从纯文本镜像改成真实 Markdown 渲染

这样用户进入文档页后，注意力会先落在 Markdown 内容本身，而不是大量输入框。

## Component Changes

### HomePage

`HomePage.tsx` 改成 dashboard 组合页：

- 顶部 `home-toolbar`：搜索条、快捷动作、仓库摘要标签
- 中部 `home-hero`：标题、说明、关键指标
- 下部 `home-grid`：快捷入口卡、最近操作、仓库状态、统计卡

### DesktopMasthead

`DesktopMasthead.tsx` 从大横幅改成轻量头部组件：

- 左侧是页面标题和上下文
- 右侧是紧凑关键指标
- 视觉更贴近“应用主区 header”而不是“着陆页 hero”

### DocumentEditor

`DocumentEditor.tsx` 会拆成三个内部层次：

- `editor-topbar`：标题、保存状态、路径操作
- `editor-layout`：正文编辑主列 + 侧边辅助列
- `metadata-panel`：前置字段与文档档案

预览区域改用 Markdown 渲染组件，支持至少：

- 标题
- 段落
- 粗体、斜体
- 列表
- 引用
- 行内代码与代码块
- 链接

## Styling Changes

`styles.css` 会做这些调整：

- 收紧输入高度、内边距和表单间距
- 为首页新增 dashboard 栅格和轻量 header 样式
- 降低 masthead 的装饰强度，提升可读性
- 调整编辑区为“主列宽、侧列窄”的工作区比例
- 新增 Markdown 预览排版，避免目前 `pre` 直接输出

## Testing Strategy

先写失败测试，再实现：

- 首页测试：确认顶部搜索、dashboard 关键标题和模块可见
- 编辑器测试：确认 Markdown 输入后，预览能显示结构化内容而不是原始标记
- 保留现有核心导航和文档编辑行为测试，避免页面重构带来回归

## Risks

- 首页布局重构可能影响现有测试对文案和结构的依赖，需要同步更新
- Markdown 渲染引入后，需要避免预览区样式失控或与深色主题冲突
- 输入区缩小后，仍要保证焦点态、可点击面积和可读性

## Success Criteria

- 首页第一屏呈现更清晰的 dashboard 结构
- 搜索入口更靠前，核心卡片与操作层级明确
- 文档页正文编辑成为绝对主工作区
- Markdown 预览不再显示原始符号，而是真实渲染结构
- 现有桌面端测试、构建和 Tauri 校验全部通过
