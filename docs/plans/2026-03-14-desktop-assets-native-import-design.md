# Desktop Assets Native Import Design

**Date:** 2026-03-14

## Goal

把桌面端资源页从“手填路径的导入表单”收敛成真正的桌面资源工作区：通过系统文件选择器导入本地文件，列表、详情和导入结果联动更新，减少资源页的割裂感。

## Current Problems

- 资源导入依赖手动输入本地路径，和桌面端能力不匹配。
- 资源页只有索引和表单，没有稳定的选中态和详情面板。
- 导入动作完成后，用户很难直接确认资源是否进入库存，也看不到关键元数据。

## Chosen Approach

保留现有三栏工作区方向，但调整职责：

- 左侧保留作用域与项目过滤。
- 中间保留资源表格列表，并支持选中资源。
- 右侧改成“详情 + 导入”面板。

导入入口改成原生文件选择：

- 点击“选择文件”打开系统文件选择器。
- 选中文件后回填路径和默认目标文件名。
- 用户补充项目 slug 后执行导入。

## UX Rules

- 不再要求用户手动输入本地文件路径。
- 当前选中的资源需要在详情区显示：
  - path
  - scope
  - project
  - size
  - sha256
- 导入成功后：
  - 刷新资源列表
  - 优先选中新导入的资源
  - 让详情区直接显示导入结果

## Data Flow

1. 页面加载时获取资源列表。
2. 用户切换作用域或项目过滤，列表重新过滤。
3. 用户点击“选择文件”，通过 Tauri dialog plugin 选择本地文件。
4. 前端从选择结果中提取默认文件名并更新表单。
5. 用户点击“导入资源”，调用现有 `import_asset` 命令。
6. 成功后重新拉取资源列表，并把选中态切到新资源。

## Testing Strategy

- 增加资源页交互测试：
  - 文件选择按钮会调用原生文件选择。
  - 选中文件后回填路径和目标文件名。
  - 导入成功后刷新列表。
  - 选中资源后详情区显示关键元数据。
- 保留现有路由和首页测试，避免资源页重构影响整体桌面壳层。

## Out of Scope

- 暂不做二进制文件缩略图预览。
- 暂不新增资源编辑或删除能力。
- 暂不修改 Rust 侧 `import_asset` 协议，只复用现有导入命令。
