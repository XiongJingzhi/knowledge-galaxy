# Desktop Create And Assets Workbench Design

## Goal

继续推进桌面端，把创建页和资源页收敛到与首页、文档页一致的桌面工作台结构，减少“单块大表单”的感觉，强化模块秩序和主次关系。

## Context

当前桌面端已经完成两步关键重构：

- 首页进入 dashboard 化的一屏总览
- 文档页形成“指挥台 + 浏览区 + 编辑区”的多面板结构

但创建页和资源页仍然偏旧：

- 创建页仍是一个大表单里串联模板、提示、字段和正文
- 资源页虽然已经分栏，但过滤、库存和导入还没有明确的桌面工作台语义

这会让二级工作页之间的质感不连续。

## Design Direction

### 1. 创建页改成三段式工作台

创建页拆成三个清晰模块：

- 左侧 `配方台`：专门放模板选择
- 中间 `创建面板`：只放当前配方上下文和关键字段
- 右侧 `正文起草`：把 Markdown 初稿输入提升成独立主工作区

这样配方选择、结构化字段和正文输入不会继续挤在同一个面板里。

### 2. 资源页改成“清单 + 导入”桌面布局

资源页保持两层逻辑：

- 左侧 `资源索引台`：过滤器、作用域提示和库存表
- 右侧 `导入面板`：只负责导入动作和目标信息

目标是让资源查看与资源导入分离，避免页面像“过滤表单 + 另一块表单”的简单拼接。

### 3. 视觉语言延续深色星系工作台

不切回普通 SaaS 白底布局，继续保留当前深色控制台方向，但收敛页面重量：

- 卡片边界更统一
- 输入区尺寸继续偏紧凑
- 标题层级更清楚
- 重要动作集中到右侧或模块底部

## Styling Changes

- 新增创建页专用 `create-workbench`、`draft-panel`、`recipe-panel` 样式
- 新增资源页专用 `asset-workbench`、`asset-index-panel`、`asset-import-panel` 样式
- 为正文起草区和导入区增加更明确的模块头部与说明区
- 保持窄屏下可退化为单列

## Testing Strategy

- 在 `apps/desktop/src/App.test.tsx` 增加两个布局回归测试
- 创建页测试确认 `配方台` 和 `正文起草` 模块存在
- 资源页测试确认 `资源索引台` 和 `导入面板` 模块存在
- 继续运行完整桌面端测试、构建和 Tauri 校验

## Success Criteria

- 创建页不再是单块长表单，而是明确的三段式工作台
- 资源页具备更清晰的索引区和导入区层次
- 页面视觉和首页、文档页更连续
- 现有行为和导航不回退
