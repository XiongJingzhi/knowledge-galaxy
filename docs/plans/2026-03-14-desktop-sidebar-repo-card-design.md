# Desktop Sidebar Repo Card Design

## Goal

把桌面端的仓库控制从主内容区迁移到侧边栏，并收掉首页里偏说明性的壳层区块，让首页只承担搜索、概览和最近动态。

## Approved Scope

### 1. 侧边栏新增仓库卡

侧边栏承接所有仓库相关操作：

- 展示当前仓库路径
- 提供目录选择 icon，点击后打开系统目录选择器并直接切换仓库
- 提供打开当前目录动作

不再保留顶部路径输入，也不提供手动编辑输入框。

### 2. 删除主区壳层

从主区移除这些块：

- 命令台
- 知识总览
- 当前页面
- 仓库状态
- 功能入口

首页不再承担“解释系统怎么工作”的职责。

### 3. 首页只保留核心工作入口

首页内容收敛为：

- 放大的全局搜索
- 概览卡
- 最近动态

并减少长段文字说明，让界面更像桌面工作台而不是后台说明页。

### 4. 二级页继续保留工作区

文档、创建、资源、校验与导出、项目页仍然保留自己的工作区布局，但整体文案继续偏短，避免多层解释。

## Component Changes

- `Sidebar.tsx` 承接仓库卡和目录操作
- `App.tsx` 去掉 `RepoSwitcher` 与 `DesktopMasthead`
- `HomePage.tsx` 删除功能入口、仓库状态和冗长 copy，放大搜索区
- `DesktopMasthead.tsx` 从主流程中移除

## Testing Strategy

- 更新 `App.test.tsx`
- 覆盖侧边栏仓库卡、目录选择切换和打开当前目录
- 确认首页不再出现命令台、知识总览、当前页面、仓库状态、功能入口
- 确认首页仍保留全局搜索、概览卡、最近动态

## Success Criteria

- 仓库控制全部迁移到侧边栏
- 首页明显变轻，搜索成为主锚点
- 已删除的壳层文案不再出现在产品里
- 所有桌面端测试、构建和 Tauri 校验通过
