# Knowledge Galaxy 桌面端操作反馈层设计

## 目标

继续推进 `apps/desktop` 的工作台体验，在现有概览层与空状态之上，补齐“操作反馈层”。

本轮不新增任何后端命令，只基于已有的前端动作结果，在界面中展示最近操作，让用户在创建、保存、导入、校验、导出、项目命令后能够立即看到结果。

## 当前缺口

桌面端现在已经有：

- 顶部概览带
- 当前视图说明
- 各工作区空状态
- 更完整的档案工作台视觉

但操作完成后的反馈仍然偏弱：

- 创建文档成功后，界面会跳转，但缺少“刚刚做成了什么”的总结
- 导入资源、保存文档、导出、校验、项目命令之后，也没有统一的成功回声
- 用户只能从局部面板推断结果，而不是从全局层面理解最近发生了什么

## 方案比较

### 方案 A：只保留局部结果面板

优点：
- 改动最小

缺点：
- 反馈分散
- 不能形成工作台层面的连续感

### 方案 B：全局 toast

优点：
- 即时
- 视觉上明显

缺点：
- 易消失
- 对桌面端这种长期工作台不够稳

### 方案 C：固定“最近操作”面板 + 局部结果保留

优点：
- 既有全局反馈，又不丢当前局部面板
- 更适合桌面工作台
- 不需要复杂状态管理

缺点：
- 需要在主界面里多占一块空间

## 采用方案

采用方案 C。

## 设计方向

新增一个固定的“最近操作 / Activity Feed”区域，放在概览带之后、各工作区之前，作为桌面端的全局反馈层。

记录项只保留最近几次操作，采用卡片化展示：

- 动作标题
- 关键对象
- 可选补充说明

示例：

- 已创建文档 / `notes/ship-note.md`
- 已保存文档 / `notes/idea.md`
- 已导入资源 / `projects/atlas/assets/hero.png`
- 校验完成 / `发现 2 个问题`
- 已导出 / `manifest`
- 已执行项目命令 / `orion · add-remote`

## 行为边界

- 只记录成功完成的动作
- 失败动作继续通过错误横幅反馈，不进入最近操作
- 最近操作按新到旧排序，限制数量，避免无限增长
- 不做持久化，本轮只保留当前会话中的反馈

## 测试策略

在 `App.test.tsx` 中新增行为测试，至少覆盖：

- 创建文档后出现最近操作记录
- 导入资源后出现最近操作记录
- 项目命令执行后出现最近操作记录

## 不做的内容

- 不做自动消失 toast
- 不做本地持久化操作历史
- 不修改 Python / Go / Rust / Tauri 后端协议
