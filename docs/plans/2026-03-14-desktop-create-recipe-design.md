# Knowledge Galaxy 桌面端创建配方台设计

## 目标

把创建中心从“通用表单”提升成“模板配方台”，让用户先选文档类型，再进入对应字段上下文。

## 当前缺口

现在创建中心已经能创建 `daily / note / decision / review / project`，但主要依赖下拉框切换：

- 不够直观
- 不同类型的重点字段没有被强调
- 首次进入创建页时，不容易理解哪种文档适合当前任务

## 方案

在创建中心顶部增加一组模板卡片，每张卡片包含：

- 类型名
- 一句用途说明
- 一个切换动作

同时在表单区顶部增加当前类型的上下文提示：

- `note`：强调标题 + 正文输入
- `daily`：强调日期
- `review`：强调标题 + 日期
- `project`：强调标题 + Git Worktree

## 交互

- 点击某张模板卡片后，`createForm.type` 立即切换
- 表单上下文说明同步更新
- 不重置用户已填写的其他字段

## 视觉方向

- 配方卡片应该有明显选择态
- 风格保持研究台 / 档案台，不做普通标签页
- 当前选中的模板要像“当前工作托盘”

## 测试策略

- 渲染创建页时显示模板配方卡片
- 点击“project 配方”后：
  - 类型切换为 `project`
  - 出现 project 对应的提示文案
  - 表单中仍能看到 `Git Worktree`
