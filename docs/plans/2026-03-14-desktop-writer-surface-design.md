# Desktop Writer Surface Design

## Context

当前桌面端文档编辑页已经完成左右工作区拆分，但左侧主编辑区仍然带有明显的“表单字段”痕迹：

- 标题输入仍然像普通字段，而不是文档标题本身
- `Markdown 编辑` 标签与正文编辑器割裂
- 标题、输入框与正文容器没有形成统一的写作表面

结果是编辑页看起来像“展示型标题 + 表单 + 大 textarea”的拼接，而不是持续写作的工作台。

## Goal

把文档编辑页左侧主区重构成一体化写作面，让标题与正文成为连续的写作流，风格更接近 Obsidian / Notion 一类的桌面写作体验。

## Options

### Option 1: Integrated Writer Surface

- 将标题输入并入写作面板
- 路径与上下文弱化成顶部说明
- `Markdown 编辑` 退成角标式提示
- 标题输入与正文编辑共用统一容器语言

优点：
- 最符合“写作面”目标
- 不改变功能与信息架构

缺点：
- 需要同时调整结构和样式

### Option 2: Style-Only Tuning

- 保持现有结构
- 仅统一字体、边框、间距

优点：
- 改动小

缺点：
- 结构割裂依然存在

### Option 3: Full Rich Editor Metaphor

- 完全仿照笔记产品的沉浸式写作区
- 顶部工具栏、块级排版、更复杂的编辑器壳层

优点：
- 体验提升最大

缺点：
- 超出当前范围

## Decision

采用 Option 1。

## Design

### Writer Surface

- 左侧主区新增 `document-writer` 容器。
- 顶部保留路径和模式信息，但弱化为上下文说明。
- 标题输入成为主写作面的第一层，使用更大的字号、更轻的字段边界和更直接的文本表现。
- `Markdown 编辑` 退成小型角标，不再以普通字段标签形式出现。
- 正文 textarea 与标题输入共用统一背景、圆角、边框与内边距节奏。

### Preview Balance

- 右侧预览区继续独立存在。
- 预览标题层级略弱于左侧写作面，避免两边都像主舞台。
- 元数据区继续置于预览下方，不改交互逻辑。

### Testing

- 在 `DocumentEditor.test.tsx` 增加一体化写作面断言：
  - 标题输入位于 `document-writer` 容器内
  - `Markdown 编辑` 使用弱化角标而非普通字段标签
- 保留保存、复制路径、预览渲染等现有测试。
