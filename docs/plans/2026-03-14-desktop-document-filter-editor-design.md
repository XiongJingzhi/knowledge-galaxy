# Desktop Document Filter And Editor Design

**Date:** 2026-03-14

## Goal

把桌面端文档工作区继续收轻：文档索引页删除占空间的分类信号区，改成搜索栏旁的 filter 入口；新建和编辑页把 Markdown 写作提升为主区，时间字段自动生成并降到次级信息层。

## Current Problems

- 文档索引页同时存在筛选表单和统计信号卡，主区高度被壳层占用过多。
- “从统计摘要一键聚焦常用分类”是解释型 UI，不是必要功能。
- 文档编辑页在 Markdown 正文之前堆了一整块元数据表单，写作不是第一视觉。
- `date`、`createdAt`、`updatedAt` 这类字段不应该让用户手动维护。

## Chosen Approach

文档索引页：

- 保留搜索和列表。
- 在搜索工具栏中增加一个 filter icon 按钮。
- 点击后弹出筛选层，承载现有筛选字段。
- 删除信号卡和整块“从统计摘要一键聚焦常用分类”说明。

文档编辑页：

- 顶部优先显示标题和 Markdown 编辑器。
- 右侧保留实时预览。
- 元数据收进次级面板，不再占据正文上方主要空间。
- 时间字段自动生成并只读展示。

## UX Rules

- 文档页默认只看到搜索、筛选入口和表格列表。
- 筛选层关闭后不影响已应用的筛选条件。
- 当前筛选条件继续通过简短摘要展示在列表顶部。
- 新建文档默认自动填充当天日期。
- 创建时间、更新时间、日期不再作为主要输入框暴露。

## Data Rules

- 新建草稿时自动设置：
  - `createdAt`
  - `updatedAt`
  - `date`
- 保存时自动刷新 `updatedAt`。
- `date` 默认跟随当天日期，但在当前 UI 中以只读元数据显示。

## Testing Strategy

- 文档页测试覆盖：
  - 页面不再渲染信号卡和解释文案
  - 搜索栏旁存在 filter 按钮
  - 点击 filter 后弹出筛选层
- 编辑页测试覆盖：
  - Markdown 编辑区在主工作区可见
  - 自动生成的时间字段以只读信息展示
  - 元数据不再作为顶部大块表单占位

## Out of Scope

- 暂不引入新的富文本或代码编辑器。
- 暂不增加高级筛选保存方案。
- 暂不开放手动覆盖自动时间字段。
