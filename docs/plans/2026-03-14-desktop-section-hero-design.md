# Knowledge Galaxy 桌面端工作区引导层设计

## 目标

在桌面端已有概览层与操作反馈层之上，再补一层“工作区引导层”，让用户进入任一导航页时，都能立即看到：

- 当前区域是做什么的
- 这一页的主路径是什么
- 下一步最常见的动作是什么

## 当前缺口

现在桌面端已经具备：

- 顶部概览带
- 最近操作反馈层
- 工作区主体内容

但当用户切换到某个 section 时，仍然需要自己从面板标题推断当前推荐动作。尤其在文档工作台、创建工作台和项目工作台里，缺少一个“强引导入口”。

## 方案比较

### 方案 A：继续只用侧栏导航 + 面板标题

优点：
- 零新增结构

缺点：
- 上下文切换弱
- 不能承接“我下一步做什么”

### 方案 B：每个 section 顶部增加一张引导 hero

优点：
- 视觉和信息层级清晰
- 能承载 section 描述与主动作
- 适合桌面工作台

缺点：
- 会占用一点垂直空间

### 方案 C：全局命令面板

优点：
- 很强

缺点：
- 本轮过大
- 交互成本高

## 采用方案

采用方案 B。

## 设计方向

在概览带和最近操作之后新增 section hero，根据当前 `section` 动态切换：

- 标题
- 一句话说明
- 两到三个高频动作按钮

### 文档工作台 hero

- 标题：文档总览台
- 描述：当前用于搜索、筛选和进入具体文档
- 动作：
  - 新建 Note
  - 重置视图

### 创建工作台 hero

- 标题：创建中心
- 描述：用于生成当前知识库里的结构化文档
- 动作：
  - 创建 Note 模板
  - 创建 Project 模板

### 资源工作台 hero

- 标题：资源中心
- 描述：管理仓库级与项目级资源
- 动作：
  - 查看全部资源
  - 切到项目级资源

### 校验与导出 hero

- 标题：校验与导出台
- 描述：检查仓库健康度并导出当前快照
- 动作：
  - 运行校验
  - 导出 manifest

### 项目工作台 hero

- 标题：项目操作台
- 描述：围绕 project 文档执行远端协作动作
- 动作：
  - 新建 Project
  - 选择第一个项目并准备操作

## 行为边界

- hero 动作优先做“切换状态”和“触发已有动作”，不新增后端命令
- hero 必须是当前工作区的快捷入口，而不是和正文内容重复
- 本轮至少保证文档工作台 hero 的两个动作可测试：
  - 新建 Note
  - 重置视图

## 测试策略

在 `App.test.tsx` 中新增：

- 点击“新建 Note”后跳到创建页，并把 `type` 预置为 `note`
- 设置搜索词与筛选后，点击“重置视图”能清空输入与过滤条件
