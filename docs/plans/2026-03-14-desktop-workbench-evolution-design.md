# Knowledge Galaxy 桌面工作台演进设计

## 目标

在不扩展后端命令面的前提下，继续推进 `apps/desktop` 的可用性与视觉完成度，让桌面端从“能用的表单集合”升级为“适合长期管理知识库的工作台”。

本轮只做桌面端前端层面的增强，保持 Tauri bridge、Python CLI、数据结构与当前仓库规范不变。

## 当前问题

基于现有代码，桌面端已经具备文档、创建、资源、校验导出、项目五个工作区，但仍有三个明显短板：

1. 缺少概览层
当前只有底部三项数字，无法快速感知知识库规模、主要状态、活跃主题或来源分布。

2. 缺少反馈层
当文档列表为空、筛选过严、尚未选择文档时，页面语义偏弱，用户只能看到留白或简单文案。

3. 视觉语言还不够成体系
当前界面更接近传统后台，结构清楚，但记忆点不足，也没有把“知识台”“档案工作区”的气质建立起来。

## 方案选择

### 方案 A：继续只补测试与交互

优点：
- 最稳
- 风险最小

缺点：
- 用户感知提升有限
- 无法回应“完善桌面端样式”的目标

### 方案 B：只做视觉重绘，不改信息结构

优点：
- 见效快
- 改动集中在 CSS

缺点：
- 只是“换皮”
- 不能解决概览与反馈缺失的问题

### 方案 C：信息架构微升级 + 视觉系统重构

优点：
- 用户能立刻看到更完整的工作台层次
- 不需要扩展后端能力
- 可以通过小规模组件和样式改造带来显著提升

缺点：
- 会同时改动 `App.tsx`、局部组件和样式

## 采用方案

采用方案 C。

## 设计方向

桌面端采用“研究台 / 档案台”方向，而不是常见 SaaS 仪表盘风格：

- 深色侧栏作为固定导航骨架
- 主工作区使用暖色纸面与高对比墨色
- 在顶部引入“知识库概览带”
- 使用更明显的分区标题、统计卡片、状态徽记和空状态说明
- 保持当前命令式工作流，但让它看起来像“正在调度知识库”，不是“在填后台”

## 本轮范围

### 1. 概览层

在主内容顶部新增概览区，展示：

- 总文档数
- 资源数
- 最近仓库数量
- `stats.groups` 中最重要的几类聚合
  - 优先展示 `type`
  - 再展示 `status`
  - 如存在再展示 `theme` 或 `source`

展示方式为可扫描卡片，而不是原先底部三段文本。

### 2. 反馈层

增强以下场景的反馈：

- 文档列表为空时，显示“无结果 / 可切换筛选 / 可去创建”的空状态面板
- 未选择文档时，右侧编辑区显示更完整的引导说明
- 当前正在使用搜索词或筛选条件时，显示“当前视图说明”
- 项目和资源面板也采用一致的空状态表达

### 3. 样式系统

保留当前暖纸色基调，但做成更鲜明的“档案工作台”：

- 更强的标题层级
- 更明确的卡片深浅关系
- 侧栏导航增加编号感和激活态
- 概览卡片引入高亮色和更强排版
- 列表项增加 hover / active 反馈
- 空状态使用大字号标题和说明文字，而不是一句平铺文案

## 测试策略

本轮先写前端测试覆盖新交互与新信息层：

- `App.test.tsx`
  - 渲染概览卡片中的统计信息
  - 文档列表为空时展示空状态
  - 搜索或过滤后展示当前视图说明

视觉样式本身不做快照测试，避免脆弱性；只验证关键文案和结构是否出现。

## 不做的内容

- 不新增桌面端后端命令
- 不修改 Python / Go / Rust CLI 行为
- 不做富文本编辑器
- 不做多窗口或多仓库并行
- 不把当前未提交的 `apps/desktop/package.json` 与 `apps/desktop/src-tauri/Cargo.toml` 变更混入本轮
