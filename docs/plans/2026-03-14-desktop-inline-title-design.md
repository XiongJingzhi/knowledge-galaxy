# Desktop Inline Title Design

## Goal

把文档编辑页的标题入口收口成一个地方，去掉左侧单独的标题输入区，让顶部的当前标题或“未命名文档”本身直接可编辑。

## Problem

当前编辑区虽然已经向统一写作面收敛，但仍然同时存在：

- 顶部工作区标题
- 左侧写作面中的标题输入

这让标题在视觉和交互上重复，也破坏了“标题 + 正文”连续写作流。

## Decision

采用“标题即输入框”的方案：

- 顶部当前标题位置直接渲染为无边框输入框。
- `未命名文档` 只作为占位提示，不再出现单独的标题编辑块。
- 左侧写作面只保留正文编辑区，标题从工作区头部进入。
- Markdown 正文继续作为主编辑区域，形成“标题在上、正文在下”的直接写作体验。

## UI Changes

- 删除左侧 `document-writer__title` 区块。
- `document-editor-title` 由静态 `h2` 改成可编辑输入。
- 顶部标题输入沿用大标题层级，但保持无边框、低干扰。
- 左侧正文区继续保留大块 Markdown 编辑面板。

## Testing

- 更新 `DocumentEditor.test.tsx`，验证标题仍可编辑，但不再通过独立标题输入块渲染。
- 验证 `document-editor-title` 对应的是输入控件，并能在修改后参与保存。
- 保留现有预览和路径复制行为测试，确保回归安全。
