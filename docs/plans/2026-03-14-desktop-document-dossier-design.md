# Knowledge Galaxy 桌面端文档档案条设计

## 目标

把当前右侧文档编辑区从“表单 + 正文”提升成真正的档案视图，让用户在编辑前先看到这篇文档的身份信息与上下文。

## 当前缺口

现在右侧编辑区虽然能改 frontmatter 和正文，但缺少一层稳定的档案信息：

- 看不到 `type / slug / createdAt / updatedAt` 的集中展示
- `theme / project / tags / source` 只有输入框，没有阅读态摘要
- 缺少一个立即可用的快捷动作，比如复制文档路径

## 方案

在编辑器头部下方增加一条“文档档案条”，包含三部分：

- 基本身份卡：类型、slug
- 时间卡：创建时间、更新时间
- 关系标签带：theme / project / tags / source 的摘要 chip

同时在右上角增加“复制路径”动作。

## 交互

- 点击“复制路径”后，把当前文档路径写入剪贴板
- 成功后按钮文案暂时变成“已复制路径”
- 切换文档后重置复制状态

## 视觉方向

- 维持“档案台 / 研究台”风格
- 不是 KPI 卡片，而是更像卡片索引与标签抽屉
- 标签带要有清晰分组，不做普通灰色 pills 堆砌

## 测试策略

- 渲染文档时展示类型、slug、创建时间、更新时间
- 点击“复制路径”后：
  - 调用剪贴板写入
  - 按钮切换为“已复制路径”
