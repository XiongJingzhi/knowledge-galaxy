# 视觉设计语言规范

> 结构先于表达。
> 
> 
> 冷静先于情绪。
> 
> 永久性先于流行。
> 

---

# 一、设计哲学

## 1.1 核心视觉隐喻

视觉必须围绕以下母题构建：

- 网格（Grid）
- 轨道（Orbit）
- 核心（Core）
- 结构线（Structural Lines）
- 深色宇宙空间（Dark Structural Space）

它们象征：

- 建模
- 秩序
- 系统
- 长期主义

禁止引入与结构无关的装饰元素。

---

## 1.2 精神气质

视觉必须体现：

- 冷静
- 克制
- 理性
- 不喧哗
- 不娱乐化

禁止：

- 夸张渐变
- 情绪化插画
- 炫技动画
- 玻璃拟态泛滥
- 彩色噪声背景

---

# 二、色彩体系

## 2.1 主背景色

深蓝黑系（主视觉底色）

建议基准：

```
--bg-primary:#0B0F14;
--bg-secondary:#111827;
```

说明：

- 永久深色主题为默认
- 不提供浅色主题
- 不做高对比纯黑

---

## 2.2 强调色（唯一行动色）

橙色为系统唯一行动信号色。

```
--accent-primary:#F97316;
--accent-hover:#FB923C;
--accent-soft:rgba(249, 115, 22, 0.15);
```

用途：

- 主按钮
- 当前激活状态
- 核心标签
- 重要交互

禁止：

- 大面积橙色铺满
- 多行动色共存

---

## 2.3 功能语义色

每种颜色必须有固定语义。

```
--info-blue:#3B82F6;/* 资产 */
--risk-red:#EF4444;/* 风险 / 集中度 */
--stable-green:#10B981;/* 流动性 / 稳定 */
--system-purple:#8B5CF6;/* 系统维度 */
```

规则：

- 不超过 5 种语义色
- 不混用语义
- 不为装饰新增颜色

---

# 三、排版规范

## 3.1 标题层级

层级必须清晰。

```
--font-hero: 64px;
--font-h1: 40px;
--font-h2: 28px;
--font-body: 16px;
--font-caption: 14px;
```

原则：

- Hero 极简、极大
- 副标题低对比
- 正文克制
- 数字优先视觉权重

---

## 3.2 语言风格

允许：

- 结构化表达
- 短句
- 抽象但克制

禁止：

- 营销语气
- 情绪煽动
- 承诺收益
- 夸张动词

---

# 四、布局规范

## 4.1 网格背景

必须保留低对比网格。

规则：

- 透明度 < 10%
- 不闪动
- 不参与交互

---

## 4.2 卡片组件

统一风格：

```
border-radius: 16px;
background:rgba(255,255,255,0.02);
border: 1pxsolidrgba(255,255,255,0.08);
```

禁止：

- 强投影
- 高光玻璃感
- 重拟态

---

## 4.3 数据模块结构

统一结构：

- 标题
- 大号数值
- 次级说明

不允许随意改为：

- 重表格
- 花式图表

数据表达优先：

清晰 > 花哨

---

# 五、按钮规范

## 5.1 主按钮

- 背景橙色
- 轻微发光
- 不跳动

```
transition:all 0.2sease;
```

---

## 5.2 次按钮

- 边框样式
- 低对比
- 不抢主视觉

---

# 六、动效规范

动效必须：

- 微弱
- 慢
- 结构化

禁止：

- 弹跳
- 粒子
- 大面积位移动画
- 娱乐游戏感效果

动效原则：

> 结构过渡，而非情绪表达。
> 

---

# 七、跨产品一致性原则

所有子产品必须：

1. 使用同一色板
2. 使用同一网格背景
3. 使用同一按钮体系
4. 使用同一导航风格
5. 使用统一字体策略

禁止：

- 单个项目独立风格实验
- 单独产品 UI 完全重构

---

# 八、对比结构原则

母站：

- 极简
- 哲学
- 宇宙结构

子产品：

- 理性
- 工具化
- 数据结构

控制台：

- 功能性优先
- 可读性优先
- 不装饰

---

# 九、设计决策过滤器

任何视觉改动必须回答：

1. 它是否增强结构感？
2. 它是否破坏冷静气质？
3. 它是否会随着时间显得廉价？

若不能同时满足，禁止修改。

---

# 十、长期约束

- 不追逐设计潮流
- 不因焦虑改 UI
- 不因“看起来高级”而增加复杂度
- 永远优先结构与可读性

---

# 终极视觉准则

> 冷静的结构感
> 
> 
> 深色的秩序空间
> 
> 克制的橙色行动信号
> 
> 长期不过时
>
